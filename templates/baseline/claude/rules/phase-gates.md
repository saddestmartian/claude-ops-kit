## Phase Gates for Large Tasks

When a task involves 3+ files or 2+ modules:

1. Complete the **read/plan phase** → report findings to the user
2. **Implement first module** → report what changed and verify
3. **Continue only after confirmation** — don't batch multiple module changes into a single silent stretch

This prevents large, hard-to-review changes and ensures the user stays informed during complex work. The cost of pausing to report is low; the cost of a wrong-direction multi-file change is high.
