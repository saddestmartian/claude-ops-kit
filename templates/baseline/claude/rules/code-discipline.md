## Code Discipline

- **Read target area before every edit.** Verify line numbers and surrounding context.
- **Surgical edits only.** One module at a time. Don't batch unrelated changes.
- **New functions should be under 60 lines.** Existing long functions are grandfathered unless actively refactoring.
- **Descriptive variable names** — no abbreviations in business logic.
- **Don't add features, refactor code, or make "improvements" beyond what was asked.** A bug fix doesn't need surrounding code cleaned up. A simple feature doesn't need extra configurability.
- **Don't add docstrings, comments, or type annotations to code you didn't change.** Only add comments where the logic isn't self-evident.
- **Verify before assuming.** Always check actual code/docs/backlog state before starting work.
