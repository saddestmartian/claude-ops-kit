## Milestone Reporting Protocol

Report progress to the user at natural milestones — not after an arbitrary tool count.

Milestones include:
- Completing a file read/analysis phase
- Finishing a module edit
- Encountering an unexpected state
- Before switching from investigation to implementation

For large tasks (3+ files), report after each discrete unit of work completes. Never go silent through an entire multi-file change — surface decisions and status as you go.
