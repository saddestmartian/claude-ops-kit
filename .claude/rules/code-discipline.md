## Code Discipline

NEVER edit code you haven't read in this session — verify line numbers and surrounding context first.

NEVER batch unrelated changes in a single edit — one module at a time, surgical only.

NEVER add features, refactoring, docstrings, comments, or type annotations beyond what was asked. A bug fix is just a bug fix.

New functions: under 60 lines. Descriptive names, no abbreviations in business logic.
