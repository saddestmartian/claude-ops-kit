## Anti-Spiral

NEVER apply a second fix on top of a failed first fix — the original assumption was wrong. STOP and re-investigate from the entry point.

NEVER continue past three "found it / wait actually" cycles — step back to first principles.

NEVER expand scope beyond 3 files or into architectural changes without discussing with the user first.

When ambiguous: ask. When stacking: stop. When guessing: research.
